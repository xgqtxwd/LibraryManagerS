# 我的第一个Java项目 - 图书管理系统 🚀

## 写在前面 📝
大家好! 这是我的第一个Java全栈项目。作为一个Java新手，从零开始构建这个项目的过程充满了挑战和乐趣。

## 为什么选择开发图书管理系统？ 🤔
说实话，最开始我也在纠结做什么项目比较好。后来想到:
- 图书管理系统涵盖了增删改查等基础功能
- 包含了用户管理、权限控制等进阶要素
- 还能加入一些有趣的功能，比如数据可视化
- 而且...作为一个爱看书的人，管理图书这件事本身就很有意思！

## 开发过程中的“有趣”经历 😅
- 第一次写后端接口时，被各种注解搞得晕头转向
- 调试跨域问题时，度过了痛并快乐的下午
- 解决Token认证时，感觉自己就是解密专家
- 实现邮件提醒功能的那一刻，仿佛打通了任督二脉
- 看到前端页面终于能正常显示数据时，开心得像个孩子

## 技术成长 🌱
通过这个项目，我：
- 从一个对Spring Boot一知半解的新手，变成了能独立开发接口的“专家”
- 从畏惧Vue的前端小白，进化成了能熟练使用Element Plus的“老手”
- 学会了使用Git管理代码（虽然有时候还是会搞混分支）
- 明白了前后端分离不是简单的“分家”，而是默契的“合作”

## 项目亮点 ✨
虽然是第一个项目，但我也加入了一些自己的思考：
- 使用邮件提醒功能，让“逾期催还”变得温柔

- 设计了响应式界面，让操作变得更加顺手
- 注重代码规范，为以后的维护和升级留下余地

# 图书管理系统

## 项目概述

这是一个基于 Spring Boot + Vue 3 开发的现代化图书管理系统。系统采用前后端分离架构,提供图书借阅、用户管理、数据统计等完整的图书管理功能。

## 系统功能

### 用户功能

用户认证

登录/注册

个人信息修改

密码重置

头像上传

2. 图书借阅

图书查询和筛选

在线借阅

图书归还

借阅历史查看

当前借阅状态

消息通知

公告查看

逾期提醒

邮件通知

### 管理员功能

用户管理

用户信息管理

用户权限控制

用户状态管理

重置用户密码

图书管理

图书信息录入

图书分类管理

库存管理

借阅状态管理

3. 系统管理

公告发布与管理

逾期记录管理

数据统计与可视化

系统日志查看

## 技术架构

### 后端技术栈

核心框架: Spring Boot

持久层: MyBatis

数据库: MySQL

认证授权: JWT Token

接口文档: Swagger

其他:

邮件服务

文件上传

全局异常处理

### 前端技术栈

核心框架: Vue 3

UI组件库: Element Plus

状态管理: Pinia

路由管理: Vue Router

HTTP客户端: Axios

其他:

数据可视化 (ECharts)

富文本编辑器

文件上传组件

## 系统特点

安全性

JWT token认证

密码加密存储

权限精细控制

防XSS攻击

易用性

响应式界面设计

人性化操作流程

丰富的提示信息

完善的错误处理

3. 可维护性

前后端分离架构

模块化设计

统一的接口规范

完整的注释文档

扩展性

松耦合设计

标准化接口

可配置化设计

支持功能扩展

## 部署要求

### 服务器环境

JDK 17+

MySQL 8.0+

Maven 3.6+

Node.js 16+

### 开发环境

IDE: IntelliJ IDEA / VS Code

数据库工具: Navicat / MySQL Workbench

API测试: Postman

## 项目亮点

完整的权限管理体系

实时的邮件提醒功能

直观的数据可视化展示

人性化的用户界面设计

完善的异常处理机制

规范的代码结构和注释

这个系统适合图书馆、学校、企业等需要进行图书资源管理的机构使用,具有较强的实用性和可扩展性。
